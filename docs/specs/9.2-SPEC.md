# 9.2 — SPEC : export client du rapport (email différé)

> Issue du GATE 1 sur l'analyse `docs/specs/9.2-ANALYSE.md`. Décision humaine
> (2026-06-04) : livrer d'abord un **export client** du rapport ; **différer le
> vrai email** tant qu'un domaine expéditeur et un rate-limit n'existent pas.

## 1. Livré dans ce ticket (export client)

Frontend uniquement, zéro backend, zéro RGPD, zéro vendor, zéro domaine.

- Bouton **« Télécharger (.md) »** dans la barre d'actions de la zone résultat,
  à côté de « Copier le rapport ».
- Réutilise l'assemblage du rapport déjà produit par `handleCopy` : extraction
  dans un helper `buildReportText()` partagé par copie ET téléchargement (pas de
  duplication).
- Génération côté navigateur via `Blob` + lien `download`
  (`analyse-cohérence-AAAA-MM-JJ.md`), `URL.revokeObjectURL` pour libérer.
- Icône `Download` ajoutée à `components/design/Icons.tsx` (style cohérent).
- Aucun contenu d'annonce brute (seulement score/verdict/piliers/actions/
  contexte local agrégé) — anti-pattern §11.3 respecté par construction.

Fichiers : `frontend/app/page.tsx`, `frontend/components/design/Icons.tsx`.

## 2. Différé (vrai email 9.2) — backlog, à reprendre via l'atelier

Le requirement 9.2 d'origine (email récapitulatif) reste pertinent mais est
bloqué par des prérequis. À reprendre quand ils seront levés. Conception déjà
arbitrée par l'analyse (à confirmer au moment venu) :

- **Prérequis EXTERNES (action humaine, bloquants prod)** :
  1. acquérir un domaine expéditeur (le projet n'en a pas, CONTEXT §3.2) ;
  2. créer le compte provider (Resend reco) et vérifier le domaine (DNS SPF/DKIM) ;
  3. poser le secret `RESEND_API_KEY` via `fly secrets`.
- **Prérequis interne** : un rate-limit (dette CONTEXT §8) — sans lui, l'endpoint
  d'envoi est un relais de spam ouvert qui peut cramer la réputation du domaine.
- **Conception cible** (reco analyse) : endpoint séparé `POST /send-report`
  recevant le résultat déjà calculé (zéro coût LLM, contrat `/analyze` intact,
  pattern `/feedback`) ; **ne rien stocker** (D1, minimisation) ; Resend via
  `requests` (zéro dépendance) ; template HTML inline + texte ; **pas d'envoi sur
  fallback LLM** ; consentement explicite (case non pré-cochée) ; mention « vos
  données ne sont pas réutilisées ».

## 3. Critères d'acceptation (export client)

1. Un bouton « Télécharger (.md) » est présent dans la zone résultat.
2. Le clic télécharge un fichier `.md` dont le contenu == celui de « Copier le
   rapport » (même `buildReportText`).
3. Aucune régression du bouton « Copier le rapport ».
4. Typecheck `tsc --noEmit` sans nouvelle erreur (hors déprécation tsconfig
   pré-existante).
5. Aucune donnée d'annonce brute dans l'export.
