## 5. OPTIONS chiffrees (choix structurants)

### OPTION A â€” Stockage : SQLite sur le volume Fly existant (table `feedback`)
- Avantages : 0 vendor, 0 secret, pattern maitrise, cout marginal nul, tests isoles deja en place. Aligne Â« MVP < 1 EUR/mois Â».
- Inconvenients : consultation = requete manuelle (SQL via `fly ssh` ou endpoint admin) ; perdu si volume detruit (idem comparables) ; mono-writer.
- **Recommandation : retenir l'option A.**

### OPTION B â€” Stockage : Supabase (Postgres manage)
- Avantages : interface de consultation prete, requetes SQL faciles.
- Inconvenients : nouveau vendor/secrets, sous-traitant RGPD, cout/lock-in, integration >> 2h. Sur-dimensionne.
- Recommandation : **non** pour le MVP.

### OPTION C â€” Formulaire externe (Tally / Google Form)
- Avantages : 0 code backend, 0 RGPD interne, le plus rapide.
- Inconvenients : vendor tiers, experience non integree, donnees hors systeme, casse le design system.
- Recommandation : **non** sauf test d'appetence en 10 min.

### OPTION (modele de donnees)
- D1 (minimal) : `id, rating (1-5), comment (text optionnel borne), created_at`. RGPD : surface minimale.
- D2 (lie a l'analyse, sans annonce) : D1 + `analysis_id` + `global_score` + `verdict` au moment du feedback. Correle satisfaction et resultat sans stocker l'annonce. RGPD : surface faible.
- D3 (lie + extrait annonce) : D2 + extrait du texte d'annonce. RGPD : surface elevee.
- Recommandation : **D2**. D1 si minimum absolu. D3 deconseille.
